1)Модуль может быть запущен с параметрами:
    -h (выводит в консоль описание параметров)

    -p (принимает путь к конфигурационному файлу, по умолчанию
        файл rule_pars.ini хранится в текущей директории проекта)

    -out (принимает путь, по которому будет создавать файлы с новостями,
          по умолчанию это патка "news" в текущей директории)

    -u (обязательный параметр, принимаюший url новостной страницы)

2) При запуске модуля с параметром -u:
    2.1) url передается в метод "site_name", в котором из адреса создается
         название новостного сайта.

    2.2) название сайта передается в метод "get_rule", в котором
         используя его ,как секцию в конфигурационном файле,
         происхоит получение параметров: правил заголовков и
         текста новостей, правило получения пути к файлу, кодировка сайта.
    
    2.3) если метод "get_rule" не обнаружит секцию с названием сайта в конфигурационном 
         файле, то он создаст секцию с новым сайтом и заполним ее пустыми пераметрами,
         далее в консоли появится сообщение о необходимости заполнить параметры
         нового сайта и повторить попытку.

    2.4) url и правило получения пути к файлу передаются в метод "get_path_to_file", 
         в котором формируется путь к файлу с расширением.

    2.5) url и кодировка сайта передаются в метод "get_page", который возвращает необработанную
         правилами страницу сайта.

    2.6) в метод "get_title", для получения заголовка статьи определенной длинны, передаются 
         страница сайта и правило получения заголовка.

    2.7) в метод "get_text", для получения текста статьи отпределенной длинны, передаются
         страница сайта и правило получения текста. Так же метод дополняет текст, url ссылками.

    2.8) в медод "write_file" передаются: путь к корневой папке с текстами новостей, путь к файлу,
         заголовок и текст новости. Метод проверяет наличие указанного пути к файлу, если его нет, 
         то создает путь. Далее создает файл по указанному пути и записывает в него данные, 
         выделяя заголовок "*" и оставляя пустую строку между абзацами.

3) При запуске модуля без параметром -u:
    
    3.1) В консоль выведется сообщение, о необходимости ввести параметр с url.
